# Feature: 자가면역 맞춤 안내문 (autoimmune-guide)

> 우선순위 정의서 기준 **E 작업**. 브랜치 `feature/autoimmune-guide` / 담당자 허승혜

## 작업 개요

자가면역 모드 사용자에게 출처 기반 맞춤 안내문을 생성하고, 사용자가 고위험 항목을
직접 체크한 경우 안내문 생성을 보류하고 "의료진 검토 필요" 상태로 표시한다.

- 선행 작업: A(REQ-KB-001/002), B(REQ-KB-003 / REQ-CHAT-007) 완료
- 효과: RAG 검색·출처 표시·NFR-SAFE-003 안전 필터 인프라를 그대로 재사용 가능

## 대상 요구사항

| REQ ID | 우선순위 | 내용 |
|--------|---------|------|
| REQ-AUTO-006 | 높음 | 고위험 플래그 — 의료진 검토 게이트 (룰 매칭) |
| REQ-GUIDE-007 | 높음 | 자가면역 안내문 — 출처 기반 생성 로직 |
| REQ-AUTO-005 | 높음 | 맞춤 안내문 — 생성 트리거·입력 정의 (오케스트레이터) |

## 구현 순서 (호출당하는 것 → 호출하는 것)

### Phase 1. REQ-AUTO-006 고위험 검토 게이트
- 사용자가 직접 체크한 위험 항목을 사전 정의 게이트 목록과 룰 매칭
- 매칭 시 "의료진 검토 필요" 상태로 안내문 생성 잠금
- LLM 미사용 / 순수 로직 → 단독 단위 테스트 가능
- 의료법 §27: 앱 자동 임상 평가·판단 금지, 사용자 체크 항목 룰 매칭만 수행

### Phase 2. REQ-GUIDE-007 안내문 생성 엔진
- RAG 검색(REQ-KB-002 재사용) + LLM(GPT-4o-mini, Temperature 0.2)
- NFR-SAFE-003 다단계 안전 필터 (REQ-CHAT-007 모듈 재사용)
- 출처 메타데이터(REQ-KB-003) + JSON Schema 검증 + HealthGuide 저장
- 고위험 플래그 수신 시 "담당 의료진 상담 권고" 단일 문구로 생성 차단

### Phase 3. REQ-AUTO-005 생성 트리거·오케스트레이터
- 트리거: 자가면역 모드 선택 + 질환 등록 + 필수 입력 충족
- 8개 입력 소스 수집 → AUTO-006 게이트 호출 → 통과 시 GUIDE-007 호출
- 재요청 정책: 입력 변경 debounce / 주 1회 자동 / 수동 요청
- 누락값 처리: 부분 안내문 생성 금지

## 입력 데이터 계약 (Stub 방식)

입력 소스 기능(REQ-AUTO-001/002, SYMP-001, ACTV-001, LAB-001, DISE-001)은
아직 미구현 상태. 입력 스키마(Pydantic)만 먼저 정의해 Stub으로 연결하고,
해당 기능 완성 후 실제 데이터 소스로 배선한다.

## 확정된 인프라

- LLM: OpenAI GPT-4o-mini (API 키 확보)
- 벡터 저장소: Qdrant (A 작업에서 구축, 사용 가능)
- 임베딩 모델: text-embedding-3-small
