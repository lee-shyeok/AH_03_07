# MediGuide (7조) — AH_03_07

## 프로젝트 개요
일반 질환자, 자가면역 질환자(루푸스, 류마티스관절염)를 위한 의료 안내 모바일 서비스. RAG 기반 챗봇과 안내문 생성 포함.

## 모노레포 구조
- `app/` — 백엔드 메인 (Python, FastAPI 추정)
- `ai_worker/` — AI 워커 (RAG, 임베딩, 챗봇)
- `infra/` — 인프라 설정
- `frontend/` — 🆕 프론트엔드 (예정: Next.js 14 App Router + TS + Tailwind + shadcn/ui)
- `docs/` — 요구사항·API 명세서·와이어프레임

## 참고 문서
- 요구사항 정의서: `docs/7조_요구사항 정의서_진짜최종.xlsx`
- API 명세서: `docs/7조_API_명세서_최종.xlsx`
- 와이어프레임: `docs/wireframes/MediGuide_Wireframe.png`

## 담당자 (허승혜)
허승혜 담당 28개 요구사항만 작업 (높음 20 / 중간 7 / 낮음 1).
상세 목록은 요구사항 정의서의 담당자="허승혜" 행 참조.

주요 요구사항:
- 프론트엔드: REQ-MODE-001/003 (모드/온보딩), REQ-DISE-001 (질환 등록), REQ-ACTV-001/002 (활성도), REQ-SYMP-001/002 (증상체크), REQ-AUTO-002/003 (약물·복약 카드)
- 백엔드/AI: REQ-KB-001/002 (RAG 임베딩·검색), REQ-CHAT-007 (자가면역 챗봇), REQ-AUTO-005 (맞춤 안내문), REQ-AUTO-006 (고위험 플래그)

## 디자인 가이드 (와이어프레임 기반)
- 모바일 우선 디자인
- 메인 컬러: 오렌지 계열
- 카드형 UI (둥근 모서리, 흰 배경)
- 경고 컬러: 노란색 배경
- 하단 네비게이션 5탭: 기록·챗봇·홈·알림·마이

## API 매핑
각 요구사항에 대응하는 API는 API 명세서의 "관련 REQ ID" 컬럼에서 확인.

## 컨벤션
- 백엔드는 기존 `app/`, `ai_worker/` 구조와 컨벤션 그대로 유지
- 프론트엔드는 `frontend/` 안에 새로 셋업 (별도 package.json)
- 프론트엔드는 백엔드 API와 연동 (개발 중에는 mock 가능)
